Card Against Reality JSON Format

This documentation describes the JSON format used to define decks of CAR. Each deck is part of a deck pack, which may contain multiple decks, each with its own set of prompt and response cards. This format allows for indefinite expansion by adding custom decks and in game customized ability.

The current version is "v":1

Custom cards are not supported in-game yet, this will be available in a future release

Structure

The JSON structure is a list that starts with a metadata object and is followed by a series of deck definitions. Each deck is represented as a nested list containing metadata, prompt cards, and response cards.

JSON Format

[
    {
        "DeckPack": true,
        "v": <version_number>
    },
    [
        {
            "Name": "<deck_name>",
            "Description": "<deck_description>",
            "v": <version_number>
        },
        [<prompt_card>, ...],
        [<response_card>, ...]
    ],
    ...
]

Elements

Deck Pack Metadata

  • DeckPack: A boolean indicating that this JSON is part of a deck pack. Always true.
  • v: An integer representing the version of the deck pack.
{
    "DeckPack": true,
    "v": 1
}

Deck Definition

Each deck is defined by a list containing three elements: metadata, prompt cards, and response cards.

Deck Metadata
  • Name: A string specifying the name of the deck.
  • Description: A string providing a description of the deck.
  • v: An integer representing the version of the deck.
{
    "Name": "Main Deck",
    "Description": "Deck of cards",
    "v": 1
}
Prompt Cards

An array of strings, where each string is a prompt card.

[
    "The President called. He needs _____________, NOW!",
    "This election day, remember: a vote for me is a vote for _____________."
]
Card response needed is determined by the amount of blank sapce _____________ multiple response is not currently supported and will be skipped
Response Cards

An array of strings, where each string is a response card.

[
    "Seymour Butts.",
    "Sharks with legs."
]

Deck Example


    [
        {
            "Name": "Main Deck",
            "Description": "Deck of cards",
            "v": 1
        },
        [],
        [
            "Yeast.",
            "YOU MUST CONSTRUCT ADDITIONAL PYLONS.",
            "Your mom.",
            "Your mum.",
            "Your weird brother."
        ]
    ]

Expansion

To add more decks, simply append additional deck definitions to the main JSON array following the same structure.

[
    {
        "DeckPack": true,
        "v": 1
    },
    [
        {
            "Name": "Main Deck",
            "Description": "Deck of cards",
            "v": 1
        },
        [],
        [
            "Yeast.",
            "YOU MUST CONSTRUCT ADDITIONAL PYLONS.",
            "Your mom.",
            "Your mum.",
            "Your weird brother."
        ]
    ],
    [
        {
            "Name": "New Deck",
            "Description": "Another set of cards",
            "v": 1
        },
        [
            "What is love?",
            "Why do birds suddenly appear?"
        ],
        [
            "Baby don't hurt me.",
            "Every time you are near."
        ]
    ]
]
